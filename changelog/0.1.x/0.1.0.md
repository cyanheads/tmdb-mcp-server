# 0.1.0 — 2026-06-13

## Added

- **`tmdb_search_titles`** — name → ranked results with integer IDs across movies, TV, and people (`multi`/`movie`/`tv`/`person` modes). The required first step before any detail lookup.
- **`tmdb_get_movie`** — full movie detail in one call via `append_to_response` (credits, videos, recommendations, similar, keywords, external IDs, US certification).
- **`tmdb_get_show`** — full TV show detail (seasons, networks, creators, content rating, plus the same appended sub-resources as movies).
- **`tmdb_get_season`** — episode list for one season with guest stars and stills.
- **`tmdb_get_person`** — person dossier with `combined_credits` filmography (cast + crew, recency-ordered) and cross-platform external IDs.
- **`tmdb_discover_titles`** — filtered discovery across movies or TV: genres, release/vote ranges, vote-count floor, cast/crew/network, watch providers + region, language, runtime, and sort.
- **`tmdb_get_trending`** — trending movies, TV, or people for the day or week.
- **`tmdb_get_watch_providers`** — region-scoped streaming availability (JustWatch-backed) with flatrate/rent/buy/ads/free provider lists and the TMDB link.
- **Resources** `tmdb://movie/{movie_id}`, `tmdb://tv/{series_id}`, `tmdb://person/{person_id}` — injectable detail context, mirroring the detail tools.
- **Image-URL resolution** — every `*_path` field resolves to a full `https://image.tmdb.org/t/p/{size}{path}` URL from the cached `/configuration` base; null paths omit the field rather than emitting a broken URL.
- **Genre-name resolution** — `genre_ids[]` on search/discover/trending results resolve to names from the cached `/genre/{movie,tv}/list` maps.
- **TMDB attribution** surfaced in every tool's output enrichment: "This product uses the TMDB API but is not endorsed or certified by TMDB."
